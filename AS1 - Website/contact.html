Reflection on my web development learning journey

Development Report

This project involved the design and development of a responsive portfolio website using HTML and CSS. The aim was to create a multi-page website that demonstrates both technical understanding and design awareness while following modern web development practices.

The website consists of five main pages: a home page, projects page, contact page, video demonstration page and a report page. Each page is connected using a consistent navigation bar, allowing users to easily move between sections of the site. I used semantic HTML elements such as header, nav, main, section and footer to structure the content appropriately. This improves both readability and accessibility while also following best practices.

In terms of design, I focused on creating a modern and clean user interface. I selected a colour scheme using blue, dark tones and white to give the website a professional appearance. To ensure consistency, I used CSS variables to define colours and reused them throughout the site. I also included a gradient hero section on each page to create a strong visual impact and improve user engagement.

For layout, I used CSS Grid to organise content across the pages. The home page uses a structured layout for introduction content, while the projects page uses a different grid layout to display project cards. These cards include images, titles and descriptions, and feature hover animations to improve interactivity. This demonstrates my understanding of layout techniques and user interface design.

A key requirement of the project was responsiveness. Initially, the website layout only worked well on larger screens. I addressed this by using media queries to adapt the layout for smaller devices. I implemented a hamburger menu that appears only on mobile screens, allowing users to toggle the navigation menu. This ensures that the website is fully usable across both desktop and mobile devices.

The contact page includes a form that uses the mailto function to allow users to send messages via their email client. While this is not the most advanced solution, it meets the requirements of the module. I also styled the form to match the rest of the website, ensuring visual consistency.

During development, I faced several challenges. One of the main difficulties was ensuring that the layout remained consistent across all pages while also adapting to different screen sizes. Debugging CSS issues, particularly with positioning and responsiveness, required careful testing and adjustments. I used validation tools to check my HTML and CSS, which helped me identify and fix errors.

Throughout the project, I also considered user experience. I added hover effects, smooth transitions and clear navigation to make the site more interactive and user-friendly. Small details such as button styling and consistent spacing helped improve the overall quality of the design.

Overall, this project helped me develop a solid understanding of HTML structure, CSS layout techniques and responsive web design. I have gained confidence in building multi-page websites and learned the importance of planning, testing and refining both the technical and visual aspects of a project. In the future, I would like to improve this website by adding JavaScript functionality and more advanced features such as dynamic content.
